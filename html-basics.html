Lesson 01

HTML Basics

Learn the language that gives every webpage its structure.

Section 1 - What is HTML?

📖 Explanation

HTML stands for HyperText Markup Language. It is the standard language used to create webpages and defines the structure of a page.

💻 Code Example

<h1>Hello World</h1> <p>My first webpage.</p>

👀 Output

💡 Tip

HTML creates the content, not the design. Use CSS to style how content looks.

Section 2 - Why HTML is Important

📖 Explanation

Without HTML a browser would have no way to know what is a heading, paragraph, link, or image. HTML is the foundation of every website.

💻 Code Example

<h1>No HTML?</h1> <p>Then there would be no headings, paragraphs, links, images, or websites.</p>

👀 Output

💡 Tip

Info

Think of HTML as the skeleton of a webpage — it gives structure and meaning.

Section 3 - HTML Document Structure

📖 Explanation

Every HTML webpage begins with the same basic structure. It tells the browser where the page starts, where information about the page is stored, and where the visible content belongs.

Don't worry if you don't understand every tag yet. You'll use this template for almost every HTML page you create.

💻 Code Example

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>My First Webpage</title> </head> <body> <h1>Hello World!</h1> <p>This is my first webpage.</p> </body> </html>

🧪 Try It Yourself

Build your first real webpage.

  1. Click Copy Code.
  2. Open Notepad or Visual Studio Code.
  3. Paste the copied code.
  4. Save the file as index.html.
  5. Double-click the file to open it in your web browser.

You should see a page with a heading saying Hello World! and a paragraph below it.

📋 Tag Reference

Tag Purpose
<!DOCTYPE html> Tells the browser this is an HTML5 document.
<html> The root element that contains the entire webpage.
<head> Stores information about the webpage that isn't displayed.
<title> Sets the text shown on the browser tab.
<body> Contains everything that appears on the webpage.

💡 Tip

Every website starts with this basic template. As you learn more HTML, you'll mostly add new elements inside the <body> section.

Section 4 - Headings

📖 Explanation

Headings (<h1> to <h6>) organize content and create a clear hierarchy. Use one <h1> per page.

💻 Code Example

<h1>Main Heading</h1> <h2>Sub Heading</h2> <h3>Small Heading</h3>

👀 Output

💡 Tip

Use headings to reflect document structure. Screen readers and search engines rely on them.

Section 5 - Paragraphs

📖 Explanation

Paragraphs (<p>) group sentences into readable blocks. Use them for normal text content.

💻 Code Example

<p>This is a paragraph.</p>

👀 Output

💡 Tip

Use paragraphs for blocks of text. Avoid using multiple <br> tags to create paragraphs.

Section 6 - Links

📖 Explanation

Links are created with the <a> tag. The href attribute sets the destination, and the link text is what users click.

💻 Code Example

<a href="https://example.com"> Visit Example </a>

👀 Output

💡 Tip

Use descriptive link text so users know where the link goes (avoid "click here").

Section 7 - Images

📖 Explanation

Images are added with the <img> tag. Use src for the image path and alt to describe the image for accessibility.

💻 Code Example

<img src="cat.jpg" alt="Cat">

👀 Output

Image preview will show here when you run the example. Replace cat.jpg with a real image path.

💡 Tip

Always include meaningful alt text for accessibility and SEO.

Section 8 - Lists

📖 Explanation

Use unordered lists (<ul>) for items without order and ordered lists (<ol>) when sequence matters.

💻 Code Example

<!-- Unordered --> <ul> <li>Apple</li> <li>Banana</li> </ul> <!-- Ordered --> <ol> <li>Wake up</li> <li>Code</li> </ol>

👀 Output

💡 Tip

Use lists for related items — they improve readability and accessibility.

Section 9 - Semantic HTML

📖 Explanation

Semantic elements give meaning to the page structure. Examples include <header>, <nav>, <main>, <section>, and <footer>.

💻 Code Example

<header>...</header> <nav>...</nav> <main> <section>...</section> </main> <footer>...</footer>

👀 Output

💡 Tip

Semantic HTML helps screen readers and search engines understand your content.
